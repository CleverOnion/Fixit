## 1. 现有布局问题诊断

### 1.1 当前布局结构分析

```
┌─────────────────────────────────────────────────────────────┐
│  Sidebar (220px)           │  Header (64px)                 │
│  #001529 深色背景          │  #FFFFFF 白色背景              │
│  ┌─────────────┐           │  ┌─────────────────────────┐   │
│  │   Logo      │           │  │[☰]      用户头像 ▼     │   │
│  ├─────────────┤           │  └─────────────────────────┘   │
│  │  🏠 首页    │           ├─────────────────────────────┤   │
│  │  📚 题库    │           │                             │   │
│  │  ✏️ 练习    │           │         页面内容              │   │
│  │  📊 统计    │           │                             │   │
│  │  ➕ 录入    │           │                             │   │
│  └─────────────┘           │                             │   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心问题清单

| 问题维度 | 具体表现 | 影响分析 |
|---------|---------|---------|
| **视觉单调** | 左侧深色 Sidebar (#001529) + 右侧白色内容区，形成强烈对比的"后台管理系统"既视感 | 缺乏产品感和高级感，用户体验停留在企业内部工具层面 |
| **空间浪费** | Sidebar 占据 220px 固定宽度，即使折叠后仍有 80px | 大屏幕下内容区被压缩，信息密度降低 |
| **导航笨重** | 5 个导航项以列表形式展示，缺乏层次感 | 无法适应功能扩展，缺乏信息分组能力 |
| **一致性差** | 各页面独立实现 Header，有的有返回按钮，有的没有 | 用户在不同页面间切换时感到困惑 |
| **交互陈旧** | 点击 Menu Item 跳转页面，无任何过渡动效 | 缺乏现代应用的流畅感和精致感 |
| **移动端不友好** | Sidebar 在移动端需要特殊处理 | 响应式适配成本高，体验割裂 |

### 1.3 用户痛点深度分析

#### 痛点一：认知负担重

- **现象**: 用户需要记住 Sidebar 中每个图标对应的功能
- **问题**: 深色背景的 Sidebar 与浅色内容区形成视觉割裂
- **影响**: 新用户需要更长的学习成本

#### 痛点二：操作路径长

- **现象**: 要执行"录入题目"需要先找到 ➕ 图标
- **问题**: 录入是高频操作，但被放在导航末位
- **影响**: 核心功能的可发现性差

#### 痛点三：氛围不专注

- **现象**: 左侧深色区域持续存在，即使不在导航
- **问题**: 干扰用户对内容区的专注
- **影响**: 难以进入"心流"状态

### 1.4 竞品导航模式对比

| 产品 | 导航模式 | 特点 | 适用场景 |
|-----|---------|------|---------|
| Linear | 左侧悬浮图标 | 极简、可收折、图标+文字hover显示 | 开发者工具、项目管理 |
| Notion | 顶部简洁导航 | 全宽、搜索主导、面包屑 | 文档协作、知识管理 |
| Raycast | 命令式导航 | 搜索驱动、快捷指令、轻量 | 效率工具、Mac原生体验 |
| Apple | 底部标签栏 | 移动端标准、大按钮、易触达 | 消费级应用 |
| 语雀 | 混合导航 | 左侧可收折、工作台入口 | 知识库、团队协作 |

**结论**: Fixit 作为**个人学习管理工具**，应借鉴 Linear + Notion 的设计哲学，追求"轻量、不喧宾夺主"的导航体验。

## 2. 导航模式分析与选择

### 2.1 候选方案评估

#### 方案 A：顶部简洁导航

```
┌─────────────────────────────────────────────────────────────────┐
│  Logo   首页   题库   练习   统计          🔍 搜索    👤 用户   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                         页面内容                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**优点**:
- 视觉清爽，内容区最大化
- 与现代 Web 应用趋势一致
- 天然支持响应式

**缺点**:
- 导航项超过 5 个时拥挤
- 缺少视觉层次
- 不利于功能扩展

**匹配度**: ⭐⭐⭐⭐

---

#### 方案 B：Mac 应用式悬浮Dock

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│                                                                 │
│                                                                 │
│                    ┌─────────────────────────────────┐         │
│                    │                                 │         │
│                    │         页面内容                │         │
│                    │                                 │         │
│                    └─────────────────────────────────┘         │
│                                                                 │
│    🏠    📚    ✏️    📊    ➕                           (浮动Dock) │
└─────────────────────────────────────────────────────────────────┘
```

**优点**:
- 极简、沉浸式
- macOS 原生体验
- 可收折、不占空间

**缺点**:
- 用户认知成本高（不像网站像App）
- 底部操作区可能被遮挡

**匹配度**: ⭐⭐⭐

---

#### 方案 C：极简侧边栏（推荐）

```
┌─────────────────────────────────────────────────────┐
│  Logo  │                                              │
│  ───── │                                              │
│  🏠    │           页面内容                            │
│  📚    │                                              │
│  ✏️    │                                              │
│  📊    │                                              │
│        │                                              │
```

**优点**:
- 轻量、可控
- 保留导航可见性
- 易于功能扩展
- 与 Linear 设计一致

**缺点**:
- 仍需占用侧边空间
- 需要处理收起状态

**匹配度**: ⭐⭐⭐⭐⭐

---

#### 方案 D：命令面板驱动（Raycast 风格）

```
┌─────────────────────────────────────────────────────────────────┐
│  Fixit                    🔍  输入命令搜索...  (Cmd+K)           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                         页面内容                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**优点**:
- 极致的简洁
- 强大的搜索能力
- 极低的视觉干扰

**缺点**:
- 用户需要记住功能入口
- 学习成本高
- 不适合中老年用户

**匹配度**: ⭐⭐⭐

### 2.2 最终方案选择

**选择方案 C：极简侧边栏 + 顶部工具栏 混合模式**

**设计理念**: "轻量导航，内容为王"

```
┌────────────────────────────────────────────────────────────────────┐
│  Fixit    首页    题库    练习    统计                 🔍   👤     │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  ┌──┐                                                           │
│  │◀│                   页面内容                                    │
│  └──┘                                                           │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

**核心决策**:

1. **侧边栏**: 仅保留 Logo + 核心导航图标（5个）
2. **顶部栏**: 搜索 + 用户头像 + 面包屑（可选）
3. **交互**: Hover 显示文字标签，收折不影响内容区
4. **动效**: 丝滑的展开/收折动画

### 2.3 方案优势分析

| 维度 | 解决方案 | 效果 |
|-----|---------|------|
| **视觉** | 浅色侧边栏 + 精致图标 | 摆脱"后台管理系统"印象 |
| **空间** | 可收折设计 + 最大内容区 | 屏幕空间利用率提升 30% |
| **扩展** | 侧边栏分组 + 二级菜单 | 支持功能迭代 |
| **交互** | Hover 展开 + 平滑过渡 | 流畅、现代的体验 |
| **移动** | 自动转换为底部导航 | 响应式零成本 |

## 3. 新导航设计方案

### 3.1 导航结构定义

#### 3.1.1 一级导航（核心功能）

| 图标 | 名称 | 路径 | 优先级 | 说明 |
|-----|------|------|-------|------|
| 🏠 | 首页 | `/` | P0 | 今日复习概览 |
| 📚 | 题库 | `/questions` | P0 | 题目管理 |
| ✏️ | 练习 | `/practice` | P0 | 答题模式 |
| 📊 | 统计 | `/stats` | P1 | 数据报告 |
| ➕ | 录入 | `/import` | P0 | 快速入口 |

#### 3.1.2 二级导航（未来扩展）

```
┌─────────────────────────────────────────────────────────┐
│  📚 题库                                               │
│  ├─ 🏠 全部题目      /questions                        │
│  ├─ 📅 按日期       /questions?sort=date               │
│  ├─ 📂 按学科       /questions?group=subject           │
│  └─ ⭐ 收藏夹       /questions?filter=favorites        │
└─────────────────────────────────────────────────────────┘
```

#### 3.1.3 工具栏元素

| 元素 | 位置 | 功能 |
|-----|------|------|
| 🔍 搜索 | 右侧 | 全局搜索（快捷键 Cmd+K） |
| 👤 用户 | 右侧 | 个人中心、设置、退出 |
| 📍 面包屑 | 左侧 | 路径导航（可选） |

### 3.2 视觉设计规范

#### 3.2.1 侧边栏尺寸

```tsx
interface NavItemProps {
```

### 3.3 交互设计

#### 3.3.1 展开/收折交互

```tsx
// 侧边栏组件逻辑
function Sidebar() {
  const [collapsed, setCollapsed] = useState(false);

  return (
    <aside className={`sidebar ${collapsed ? 'collapsed' : ''}`}>
      {/* Logo 区域 */}
      <div className="logo">
        {collapsed ? (
          <span className="logo-icon">F</span>
        ) : (
          <span className="logo-text">Fixit</span>
        )}
      </div>

      {/* 导航项 */}
      <nav className="nav-items">
        {navItems.map(item => (
          <Tooltip
            key={item.path}
            title={collapsed ? item.label : ''}
            placement="right"
          >
            <NavItem {...item} />
          </Tooltip>
        ))}
      </nav>

      {/* 底部：设置/收折 */}
      <div className="sidebar-footer">
        <button
          className="collapse-btn"
          onClick={() => setCollapsed(!collapsed)}
        >
          {collapsed ? <MenuUnfoldOutlined /> : <MenuFoldOutlined />}
        </button>
      </div>
    </aside>
  );
}
```

#### 3.3.2 快捷键支持

| 快捷键 | 功能 |
|-------|------|
| `Cmd/Ctrl + K` | 打开搜索面板 |
| `Cmd/Ctrl + \` | 收折/展开侧边栏 |
| `1-5` | 快速跳转对应导航项 |

## 6. 用户流程优化

### 6.1 核心用户旅程

#### 旅程一：每日复习流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                         用户目标: 完成今日复习                       │
└─────────────────────────────────────────────────────────────────────┘

登录 → 首页 → 点击"开始复习" → 练习页 → 完成 → 离开

优化前:
1. 用户登录 (2s)
2. 等待页面加载 (1s)
3. 找到并点击"练习"导航 (3s)
4. 等待练习页加载 (1s)
5. 开始答题 (5min)
总计: ~7s + 5min

优化后:
1. 用户登录 (2s)
2. 首页直接显示"开始复习"按钮 (1s)
3. 一键进入练习 (0.5s)
4. 开始答题 (5min)
总计: ~3.5s + 5min

节省时间: 50%
```

#### 旅程二：录入新题流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                       用户目标: 快速录入错题                          │
└─────────────────────────────────────────────────────────────────────┘

发现错题 → 点击"录入" → 拍照/输入 → 保存 → 完成

优化设计:
1. 侧边栏"+"图标醒目
2. 录入页支持图片直接识别
3. 表单即时验证
4. 保存后自动返回
```

### 6.2 关键流程优化点

| 流程 | 痛点 | 优化方案 |
|-----|------|---------|
| 每日复习 | 需要寻找导航入口 | 首页突出"开始复习"按钮 |
| 录入题目 | 流程繁琐 | AI 识别 + 快捷录入 |
| 查看统计 | 信息分散 | 首页聚合关键数据 |
| 搜索题目 | 功能隐蔽 | 全局搜索 (Cmd+K) |

### 6.3 快捷键设计

| 快捷键 | 功能 | 适用场景 |
|-------|------|---------|

| `Cmd/Ctrl + \` | 收折侧边栏 | 专注阅读 |
| `1` | 跳转到首页 | 快速导航 |
| `2` | 跳转到题库 | 快速导航 |
